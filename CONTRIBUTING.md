# Contributing to Steel-Defect-Ai

We love your input! We want to make contributing to this project as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## 🚀 Our Development Process

1. **Fork** the repo and create your branch from `main`.
2. **Setup** the environment following the [README](README.md).
3. **Commit** your changes with clear, descriptive messages.
4. **Push** to your fork.
5. **Open a Pull Request**.

## 🎨 Style Guidelines

- **Python**: Follow [PEP 8](https://www.python.org/dev/peps/pep-0008/). Use docstrings for all functions.
- **Javascript/React**: Use functional components and hooks. Follow the existing project aesthetic (Inter/JetBrains Mono fonts).
- **CSS**: Use variables for colors and spacing.

## 🐛 Reporting Bugs

When reporting a bug, please include:
- A clear title and description.
- Steps to reproduce.
- Expected vs Actual behavior.
- Screenshots if applicable.
